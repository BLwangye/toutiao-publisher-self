# 种子评论独立命令 — 设计文档

**日期**: 2026-06-06
**目标**: 为自己的已发布文章自动生成并提交有吸引力的种子评论。

## 功能概述

新增 `--self-comment` CLI 选项，独立于发布流程。程序打开已发布文章列表，用户选择目标文章，DeepSeek 根据文章内容自动选择"争议提问"或"金句共鸣"策略生成评论，直接提交。

## 使用方式

```bash
npx tsx src/cli.ts --self-comment
```

## 处理流程

1. **连接 + 登录** — 连接 CDP 浏览器，验证 mp.toutiao.com 登录状态
2. **打开文章列表** — `https://mp.toutiao.com/profile_v4/graphic/articles`
3. **抓取文章列表** — 页面中提取已发布文章的标题和链接
4. **用户选择** — 终端打印编号列表，用户输入数字选择
5. **抓取文章内容** — 跳转到选中文章，提取正文内容
6. **生成种子评论** — DeepSeek 根据文章内容，在以下策略中自动选择或结合：
   - **争议提问**: 抛出一个让人忍不住想回答的问题
   - **金句共鸣**: 提炼文章中最扎心的点，用有共鸣的方式表达
7. **提交评论** — 复用 `typeComment` 逻辑，在文章评论区提交

## DeepSeek Prompt 设计

种子评论 prompt 与 interact 中的中性评论 prompt 完全不同：

```
根据以下文章内容，写一条能引发互动的种子评论（严格60-120字）。

策略（根据文章内容自动选择最合适的一种或结合使用）：
- **争议提问**：提炼文中一个存在争议或值得深入的点，用一个引人思考的问题表达，让人忍不住想回答
- **金句共鸣**：把文中最扎心的一句话或观点用大白话重新说出来，加上简短的感叹，让读者觉得"说的就是我"

要求：
- 语气真实，像真实读者的有感而发，不官方、不客气
- 不要用"写得很好""文章不错"这种客套话
- 不要引用原文（别说"文中提到..."）
- 要有互动感——让人觉得不回复都难受
- 即使是赞同的观点，也要表达得有张力

标题：{title}
正文：{content}

请只输出评论内容，不要有任何前缀或引号。
```

## 错误处理

| 场景 | 处理 |
|------|------|
| 文章列表为空 | 提示"暂无已发布文章"，退出 |
| 用户选择无效编号 | 提示重新输入 |
| 文章正文抓取失败（如视频类） | 降级为仅基于标题生成评论 |
| 评论提交失败（按钮禁用等） | 重试最多 3 次，仍失败则报告 |
| 评论区未加载 | 等待 5s 重试，仍无则退出 |
